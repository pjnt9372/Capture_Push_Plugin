# 🔍 `config.ini` 各部分详解

## 📌 `[logging]`

**作用**：配置应用程序的日志级别

| 配置项     | 值                     | 说明                                        |
|-----------|-----------------------|-------------------------------------------|
| `level`   | `DEBUG`  <br/> `INFO` <br/> `WARNING` <br/> `ERROR` <br/> `CRITICAL` | `DEBUG` 包含最详细的日志信息，主要用于调试<br/> `INFO` 一般信息，确认程序按预期工作<br/> `WARNING` 表示潜在的问题，但程序仍可继续运行<br/> `ERROR` 表示错误事件，程序某些功能无法执行<br/> `CRITICAL` 表示严重错误，程序可能无法继续运行 |

## 📌 `[run_model]`

**作用**：避免开发时重复拉取造成封禁

| 配置项     | 值                     | 说明                                        |
|-----------|-----------------------|-------------------------------------------|
| `model`   | `DEV`  <br/> `BUILD` | `DEV` 处于开发模式不反复拉取<br/> `BUILD` 构建模式正常拉取 |

## 📌 `[push]`

**作用**：配置消息推送方式（用户有且只能同时启用一种推送方式）

| 配置项     | 值                     | 说明                                        |
|-----------|-----------------------|-------------------------------------------|
| `method`  | `none` <br/> `email` <br/> `test1` <br/> `wechat` <br/> `dingtalk` <br/> `telegram` | `none` 不启用任何推送<br/> `email` 邮件推送（需配置 `[email]` 节）<br/> `test1` TEST1推送（需配置 `[test1]` 节，预留接口）<br/> `wechat` 微信推送（未来扩展）<br/> `dingtalk` 钉钉推送（未来扩展）<br/> `telegram` Telegram推送（未来扩展） |

**扩展指南**：添加新推送方式请参考 `core/senders/EXTENSION_GUIDE.md`

## 📌 `[email]`

**作用**：邮件推送的配置信息（当 `[push].method=email` 时生效）

| 配置项     | 说明                                        |
|-----------|-------------------------------------------|
| `smtp`    | SMTP 服务器地址（如 smtp.qq.com）          |
| `port`    | SMTP 端口（通常为 465 或 587）             |
| `sender`  | 发件人邮箱地址                              |
| `receiver`| 收件人邮箱地址                              |
| `auth`    | 邮箱授权码（非邮箱登录密码）                |

## 📌 `[test1]`

**作用**：TEST1 推送的配置信息（当 `[push].method=test1` 时生效）

**说明**：TEST1 是预留的推送方式接口，用于测试和开发新的推送实现。参数名称和用途可根据实际推送方式自定义。

| 配置项     | 说明                                        |
|-----------|-------------------------------------------|
| `param1`  | 自定义参数1（如：服务地址、AppID 等）        |
| `param2`  | 自定义参数2（如：用户ID、频道ID 等）         |
| `param3`  | 密钥/Token（敏感信息，建议加密存储）          |
